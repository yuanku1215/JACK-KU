Overview

This capstone internship is a two-month, full-time summer program bridging my junior spring and senior year. Unlike a single-assignment course, the capstone mirrors real-world geomatics practice: we start from field data acquisition, ensure quality control, then build a complete mapping product that is usable for analysis and communication.

Our goal was to produce a campus-wide 3D map as the foundational layer, and then overlay multiple thematic GIS maps to support different use cases (navigation, management, and spatial understanding). The project required combining traditional geodetic surveying instruments (level, total station, GNSS) with photogrammetric modeling and GIS integration.

What makes it capstone-level: it is end-to-end (field ‚Üí processing ‚Üí product), requires accuracy awareness, and forces consistent coordinate control so that 3D models and GIS layers align in the same spatial reference frame.

Learning Goals

Field Data Collection

Plan and execute measurements using leveling, total station, and GNSS with clear quality-control checkpoints.

3D Modeling

Apply photogrammetry workflow to reconstruct 3D geometry, then align it with control in a consistent reference system.

GIS Integration

Build thematic layers and cartographic outputs on top of the 3D base map for practical campus-scale applications.

Workflow: Field ‚Üí Processing ‚Üí Delivery

The project pipeline emphasizes traceability: each stage is documented, checked, and connected to the final map. This structure is also how professional surveying and mapping projects ensure reliability.

1) Planning & Control

Define the working coordinate reference, prepare control points, and decide survey routes to ensure consistent alignment across instruments and photogrammetry.

2) Field Survey

Collect elevation differences via leveling, measure detailed geometry with total station, and establish positioning with GNSS for robust control and georeferencing.

3) Photogrammetry & 3D Reconstruction

Process image sets into a 3D model (point cloud / mesh) and refine alignment using ground control and quality checks to reduce systematic distortions.

4) GIS Thematic Mapping

Build multiple thematic layers (e.g., infrastructure, facilities, accessibility, or functional zoning), then produce maps that can be used for communication and spatial analysis.

Quality control (QC) mindset used in this capstone
We treated the project as a measurement system, not only a visualization task. QC was embedded in every stage:
  • Consistency: ensure all outputs share the same coordinate reference and control network.
  • Redundancy: design field measurements with checks (e.g., loops, repeated observations).
  • Error awareness: monitor residuals and alignment errors during photogrammetry and control fitting.
  • Deliverable-ready: maps must be interpretable, reproducible, and documented.

What I Learned

  • End-to-end thinking: field data quality directly controls the credibility of final visual outputs.
  • Coordinate consistency: integrating surveying, photogrammetry, and GIS requires strict reference control.
  • Measurement mindset: good maps come from validated measurements and transparent QC, not only nice rendering.
  • Team workflow: a reliable data pipeline matters as much as any single algorithm or tool.

Extensions & Next Steps
